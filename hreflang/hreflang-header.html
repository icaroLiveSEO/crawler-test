Teste de Hreflang via HTTP Header

Esta página usa hreflang via HTTP Header

O servidor está configurado para enviar os seguintes cabeçalhos HTTP Link com esta página:

Link: <https://exemplo.com.br/hreflang-header.html>; rel="alternate"; hreflang="pt-br"
Link: <https://example.com/hreflang-header.html>; rel="alternate"; hreflang="en-us"
Link: <https://ejemplo.com/hreflang-header.html>; rel="alternate"; hreflang="es"
Link: <https://example.com/hreflang-header.html>; rel="alternate"; hreflang="x-default"
Nota: Para que este teste funcione corretamente, o servidor web deve estar configurado para enviar os cabeçalhos HTTP Link com esta página. Isso geralmente requer configuração no servidor web (Apache, Nginx, etc.) ou em um arquivo .htaccess.

Sobre este teste

Esta página demonstra o uso de cabeçalhos HTTP Link com atributos hreflang em vez de tags link no HTML. Ambos têm o mesmo efeito, mas os cabeçalhos HTTP podem ser aplicados a qualquer tipo de arquivo, não apenas a páginas HTML.

O que o crawler deve fazer:

  • Detectar os cabeçalhos HTTP Link com atributos hreflang
  • Identificar os idiomas e regiões especificados
  • Associar cada URL com o idioma/região correspondente

Vantagens do hreflang via HTTP Header:

  • Pode ser aplicado a qualquer tipo de arquivo (PDF, imagens, etc.)
  • Útil para arquivos não-HTML que precisam de indicação de idioma
  • Pode ser configurado em nível de servidor para grupos de arquivos

Configuração de exemplo para Apache (.htaccess):

<Files "hreflang-header.html">
    Header add Link "<https://exemplo.com.br/hreflang-header.html>; rel=\"alternate\"; hreflang=\"pt-br\""
    Header add Link "<https://example.com/hreflang-header.html>; rel=\"alternate\"; hreflang=\"en-us\""
    Header add Link "<https://ejemplo.com/hreflang-header.html>; rel=\"alternate\"; hreflang=\"es\""
    Header add Link "<https://example.com/hreflang-header.html>; rel=\"alternate\"; hreflang=\"x-default\""
</Files>
