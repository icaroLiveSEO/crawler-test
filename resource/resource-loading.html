Teste de Carregamento de Recursos

Sobre o Carregamento de Recursos

Esta página testa como o crawler lida com diferentes tipos de recursos (CSS, JavaScript) que têm comportamentos variados de carregamento, incluindo recursos que carregam normalmente, recursos que carregam lentamente e recursos que falham ao carregar.

Recursos Carregados

Tipo URL Status Descrição
CSS css/styles.css OK Arquivo CSS válido com carregamento normal
CSS css/non-existent.css Erro 404 Arquivo CSS inexistente (URL inválida)
CSS css/slow-loading.css Lento Arquivo CSS com carregamento lento (simulado)
JavaScript js/script.js OK Arquivo JavaScript válido com carregamento normal
JavaScript js/non-existent.js Erro 404 Arquivo JavaScript inexistente (URL inválida)
JavaScript js/slow-loading.js Lento Arquivo JavaScript com carregamento lento (simulado)

Código HTML

Abaixo está o código HTML usado para carregar os recursos nesta página:

<!-- CSS válido com carregamento normal --> <link rel="stylesheet" href="css/styles.css"> <!-- CSS com URL inválida --> <link rel="stylesheet" href="css/non-existent.css"> <!-- CSS com carregamento lento (simulado) --> <link rel="stylesheet" href="css/slow-loading.css"> <!-- JavaScript válido com carregamento normal --> <script src="js/script.js"></script> <!-- JavaScript com URL inválida --> <script src="js/non-existent.js"></script> <!-- JavaScript com carregamento lento (simulado) --> <script src="js/slow-loading.js" defer></script>

Sobre este teste

Esta página testa como o crawler lida com diferentes tipos de recursos e seus comportamentos de carregamento.

O que o crawler deve fazer:

  • Detectar todos os recursos externos carregados pela página
  • Identificar recursos que falham ao carregar (URLs inválidas)
  • Possivelmente detectar recursos com carregamento lento
  • Analisar o impacto dos recursos no desempenho da página
  • Verificar se há recursos bloqueando o renderização da página
